Búsqueda en el World Wide We 

UNA BRÚJULA PARA EL WEB 

Un agente inteligente emplea tecnología vanguardista de «metabúsqueda» para indagar información en el Web y organizar los resultados. 

REX BALDAZO 

Investigar un tema concreto en el World Wide Web es muy parecido a viajar en coche sin un mapa de carreteras: el viaje puede ser muy interesante porque, quizás, nunca se llegará al destino.
Los servicios de índice como Lycos ( www.lycos.com ), OpenText ( www.opentext.com ) son un buen comienzo, pero ningún índice puede cubrirlo todo ni puede estar siempre actualizado; muchas veces el navegante de Internet acaba consultando la misma información en varios de estos servicios.
La nueva utilidad Quarterdeck WebCompass para Windows simplifica mucho la tarea de buscar en múltiples índices. 
Desarrollado por Limbex, WebCompass no es tanto una herramienta de búsqueda como una utilidad de metabúsqueda (véase el recuadro Claves tecnológicas ).
Esto significa que WebCompass no realiza las búsquedas por sí mismo, sino que acude a otros recursos como Lycos, Yahoo ( www.yahoo.com ) o el servidor de BYTE ( www.byte.com ).
WebCompass fue galardonado por los redactores de BYTE con el premio al mejor producto del salón Comdex de primavera del año pasado y, tras someter esta aplicación a unas pruebas exhaustivas, nuestra opinión sobre ella ha mejorado aún más.

Con cualquier navegador 

El sistema que ejecuta WebCompass es una aplicación CGI (Interfaz Común de Pasarela), de modo que hace falta ejecutar un servidor Web en la máquina local; (a este fin, WebCompass incluye una versión de Quarterdeck WebServer).

El usuario puede utilizar el navegador Web que desee para controlar WebCompass (véase la pantalla superior) y, aunque el paquete de WebCompass incluye el browser QMosaic de Quarterdeck, no hemos podido utilizarlo debido a problemas de configuración con nuestra conexión Internet (a través de Microsoft Network).

Por este motivo, hemos empleado Microsoft Internet Explorer 2.0 como interfaz para WebCompass, con lo que hemos podido comprobar el alto grado de flexibilidad de la WebCompass: no importa el navegador que se emplee con esta aplicación.

WebCompass también incluye una base de datos de formato Microsoft Access 2.0, así como controladores ODBC (Conectividad Abierta de Bases de Datos) para almacenar e indexar resultados de búsquedas.
Al configurar WebCompass, se puede instalar una base de datos que alberga información sobre diversos temas; esta base de datos prepoblada resulta de mucha ayuda para organizar datos y establecer referencias cruzadas entre ellos.

Al sumar todos estos elementos, WebCompass adquiere un volumen imponente en el disco.
Quarterdeck recomienda reservar unos 30 MB de espacio en disco para una instalación típica, la mitad de los cuales corresponde a la base de datos con sus propios datos ya introducidos.
Cuando el usuario empiece a almacenar e indexar los resultados de sus búsquedas, el tamaño de la base crecerá inexorablemente.

Gestión de recursos 

La clave de WebCompass es su capacidad para interactuar con distintos mecanismos de búsqueda, que en la terminología de Quarterdeck se denominan «recursos».
El paquete incluye varios recursos preconfigurados, incluyendo CNN, CNN Sports, Yahoo, Lycos y Excite; cada recurso está agrupado en una categoría, como Recursos Generales o Recursos Técnicos.

Cuando se efectúa una búsqueda WebCompass, el usuario especifica los términos de búsqueda y los recursos que desea utilizar.
Así, WebCompass se ocupa del trabajo sucio: consulta los recursos, recopila las respuestas y las presenta en una única página (véase la pantalla inferior).

Se pueden utilizar categorías para agrupar recursos similares.
Por ejemplo, si se investiga sobre el término Montana , pero referido sólo al jugador de fútbol norteamericano (y no sobre el Estado norteamericano), se puede especificar a WebCompass que sólo utilice los recursos del grupo Deportes.
Los resultados de la búsqueda incluirán referencias a Joe Montana, a la conferencia Big Sky y, quizás, al Parque Nacional de Yellowstone.

Si los recursos preconfigurados no son suficientes, se pueden añadir otros nuevos y reasignar los existentes a distintas categorías.
El proceso de agregar un recurso nuevo se divide en dos pasos.
Primero se introduce la dirección URL, información descriptiva y la categoría deseada, y después WebCompass lee el recurso situado en esta URL.
Hay que especificar el campo de la página de recursos donde se introducirá el texto a buscar, y también se deben introducir criterios de búsqueda para configurar la página del recurso de búsqueda.
Éste se agrega a una categoría o grupo de recursos, y WebCompass lo podrá consultar de modo automático siempre que se utilice dicho grupo.

Agente encubierto 

WebCompass tiene dos formas de búsqueda: interactiva y basada en agente.
La primera corresponde al modo tradicional de búsqueda: el usuario teclea los términos a buscar, selecciona una categoría de recursos y lanza la búsqueda.
En este modo, WebCompass realiza las pesquisas y presenta en una sola página todos los resultados obtenidos de los diversos recursos consultados.

Se puede investigar cada uno de los resultados y añadirlo a la base de datos local marcando la casilla correspondiente en el formulario.
Este resultado se puede incluir en un grupo temático que, en general, estará relacionado con el término de búsqueda empleado para encontrarlo.
El Agente WebCompass recuperará e indexará el documento en la base de datos local para tenerlo como referencia futura.
De hecho, esta base de datos local es una de las categorías de búsqueda de WebCompass, que el programa puede rastrear como cualquier otra categoría.

El Agente es una de las características exclusivas de WebCompass.
Se ejecuta como una aplicación externa separada del navegador, aunque se activa desde la página principal de WebCompass.
El Agente tiene dos funciones: recupera e indexa artículos que el usuario ha seleccionado como resultado de una búsqueda previa, y automáticamente realiza consultas por el usuario.

Cuando ya se han especificado los artículos a añadir a la base de datos local, el Agente puede transferirlos al ordenador del usuario, indexarlos y organizar los documentos para una inspección posterior.
El sistema de indexación es bastante fiable y, en general, consigue agrupar los artículos relacionados.
Durante las pruebas de WebCompass, el Agente ha agrupado en alguna ocasión artículos no relacionados entre sí, si bien en general la primera organización de documentos ha sido razonable.

Cuando se realiza una búsqueda de forma interactiva, los resultados se pueden añadir a un grupo temático nuevo o a uno existente.
Hemos ejecutado una búsqueda con la palabra Bosnia y creado un nuevo tema, también denominado Bosnia (los temas son palabras clave que ayudan a organizar la información en la base de datos local).
Si se especifica que un tema determinado es activo , el Agente WebCompass ejecutará de modo automático las búsquedas relacionadas con el tema.
Después, el software añade los nuevos resultados a la base de datos local para una inspección posterior.
Mientras se realizan las búsquedas del Agente, el usuario puede continuar utilizando su navegador Web para visualizar otras informaciones.

Algunos inconvenientes 

Si el Agente repite la misma consulta varios días seguidos, el número de artículos indexados en la base de datos local crecerá rápidamente.
Por suerte, WebCompass incluye herramientas para gestionar la base de datos, aunque a veces éstas resultan algo torpes porque también están basadas en Web.
Por ejemplo, se puede eliminar un grupo de artículos con un único clic, si bien hay que saltar a otro formulario para eliminar un único artículo.
Se pueden trasladar artículos de un grupo temático a otro, pero los elementos no se pueden arrastrar y soltar con el ratón porque la interfaz de la herramienta está basada en Web.
En vez del ratón, hay que trabajar con un formulario HTML (Lenguaje de Marcas de Hipertexto), que también es obligatorio para configurar los nuevos recursos y el Agente.

La capacidad para emplear diversos navegadores Web es, al mismo tiempo, un punto fuerte y un punto débil de WebCompass.
El sistema de caché de disco de algunos navegadores puede provocar que la pantalla del navegador no muestre lo que WebCompass está haciendo.
Por ejemplo, el navegador puede indicar que el Agente está inactivo cuando, en realidad, anda suelto por Internet indexando documentos.
Esta inconsistencia puede solucionarse volviendo a cargar el navegador siempre que haya sospechas de que puede haber perdido la sincronización con WebCompass.

Lo más molesto de WebCompass es que es una aplicación monousuario, lo que impide que diversos usuarios puedan mantener distintas bases de datos locales por medio de una instalación común en red local.

Sin embargo, estos fallos son poco importantes y, sin duda, en su mayoría quedarán corregidos en la próxima edición del paquete.
Lo que cuenta es que WebCompass ha pasado a formar parte imprescindible de nuestro juego de herramientas Internet.

recuadro página 98 CLAVES TECNOLÓGICAS Metabúsqueda: más allá de la búsqueda

WebCompass es una herramienta de metabúsqueda , es decir, un programa que puede pasar consultas a otros recursos de búsqueda y procesa los resultados.
Así, WebCompass aumenta el valor de los sistemas de búsqueda, pero depende de ninguno de ellos en particular.

La primera parte de la metabúsqueda es la interacción con recursos de búsqueda como, por ejemplo, Yahoo.
WebCompass hace, básicamente, lo mismo que haría un usuario humano: introduce los términos de búsqueda y los transmite a Yahoo.
Después utiliza distintas técnicas de inteligencia artificial para extraer expresiones nominales, que luego clasifica y ordena por medio de una combinación de reglas estadísticas y heurísticas.
Por ejemplo, el Agente puede guiarse por la frecuencia con la que aparece una expresión nominal (un método estadístico) u otorgar mayor importancia a otra expresión porque aparece en la primera frase de un párrafo (un método heurístico).

El Agente utiliza las expresiones nominales para componer un resumen de cada documento.
En la base de datos local se almacena este resumen (en lugar del documento completo) para referencias futuras.
Cuando deja de ser útil, un resumen se puede borrar de la base de datos.

WebCompass se basa en las clasificaciones de la base de datos para agrupar documentos similares por medio de la asociación conceptual , otra técnica de inteligencia artificial.
Cuando el Agente ha determinado los documentos que guardan algún parecido, analiza las similitudes y produce un título para describir al grupo de documentos.

Este título toma la forma de un hiperenlace que sirve para saltar de un grupo a otro de documentos relacionados.

Dentro de un tiempo, WebCompass podría incluir una técnica de inteligencia artificial que aplica el test de Turing.
Los esfuerzos de los investigadores de inteligencia artificial están dando frutos en productos que, como WebCompass, están basados en agentes.
