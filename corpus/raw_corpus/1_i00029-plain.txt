
OPTIMIZADOS PARA EJECUTAR APPLETS JAVA 

Los chips Java aceleran los programas applet 

Tres nuevos microprocesadores de Sun Microelectronics (empresa antes denominada Sparc Technology Business) son las tres primeras CPU dedicadas a ejecutar software Java.

Estos chips han sido diseñados para ejecutar programas Java a una velocidad mucho mayor que de un sistema Java basado en software e implementado sobre un microprocesador de propósito general, como los x86, PowerPC o Sparc.

El objetivo de Sun es aumentar el rendimiento de Java y facilitar a los fabricantes el diseño de dispositivos Java de bajo precio como, por ejemplo, equipos Web PC y productos mixtos (teléfonos celulares y asistentes digitales personales).

Los programadores escriben applets con el lenguaje de programación Java, que se parece mucho a C++.
Un compilador Java traduce este código fuente a un formato compacto conocido como bytecode ; éste es más compacto y seguro, y su ejecución es más eficaz que la del código fuente en bruto.
El bytecode se ejecuta en un nivel de software denominado motor virtual Java , que contiene un intérprete de ejecución que traduce el bytecode a código nativo del microprocesador subyacente.
El bytecode de Java es análogo al código binario ejecutable, excepto en que no es específico de una arquitectura del microprocesador; gracias a esta característica, los applets de Java pueden ejecutarse en cualquier ordenador que disponga de un motor virtual Java.

A diferencia del código binario compilado, los applets de Java no se traducen a código máquina nativo hasta el momento de su ejecución.
El inconveniente técnico de este enfoque estriba en que el proceso de interpretación en tiempo real consume tiempo de procesador e incide sobre el rendimiento de forma negativa.
Con los chips Java de Sun no es necesario efectuar una interpretación durante el proceso de ejecución, ya que estos chips ejecutan el bytecode directamente.
De hecho, el bytecode de Java constituye el conjunto de instrucciones nativas de los microprocesadores Java.

Uno de los tres productos anunciados recientemente por Sun constituye realmente el núcleo de una CPU; Sun desea ofrecer la licencia de este producto a otros fabricantes de chips.

Conocido como picoJava, se trata de un núcleo extremadamente pequeño (no tiene más de 25 milímetros cuadrados) que los licenciatarios pueden configurar.
Según Sun, los chips Java de gama baja basados en el picoJava podrían costar menos de 25 dólares.

MicroJava es un microcontrolador de Sun basado en el núcleo picoJava.
Tiene un tamaño de unos 50 milímetros cuadrados y está pensado para equipos de telecomunicaciones y otras aplicaciones mixtas.
Sun espera disponer de muestras de este chip a principios de 1997 y su precio podría estar situado entre los 25 y 50 dólares.

En la gama alta encontramos el procesador ultraJava, que es de tres a cinco veces más rápido que el microJava.

Dispone de extensiones multimedia parecidas a las de los procesadores Sun UltraSparc utilizados en estaciones de trabajo.
Entre otras cosas, estas extensiones permiten gráficos 3-D rápidos.
El ultraJava ha sido diseñado pensando en los futuros equipos Web PC y en otros dispositivos multimedia semejantes.
Sun espera disponer de muestras de este chip a finales de 1997 y su precio podría rondar los 100 dólares.

La mayor desventaja de los procesadores Java consiste en que sólo pueden ejecutar software Java.
Sin embargo, además de hacer posible la existencia de dispositivos Java dedicados, también podrían servir como coprocesadores Java de alta velocidad en ordenadores personales de propósito general.
